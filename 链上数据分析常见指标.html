链上数据分析实战

Chapter 4 链上数据分析常见指标

我们在这里梳理一下常用的指标.

  1. 用户行为
  • 活动地址数
  • 新增地址数量
  • 持仓地址数 地址分布 : 持币者的财富分布, 是否存在巨鲸
  1. 资金流动
  • 资金流入 : 流入协议或地址资金量
  • 资金流出 : 流出协议或者地址大的资金量
  • 资金净流量 : 流入 - 流出, 判断市场趋势
  • 巨鲸交易 : 单笔交易金额超过阈值
  1. 智能合约交互
  • 合约调用次数
  • 活跃智能合约 : 一定时间内被调用的智能合约数量
  • Gas 消耗量
  • NFT铸造量
  • DeFi交互量
  1. 市场表现
  • TVL : DeFi 协议中锁定的总价值
  • 总市值
  • 代币持仓分布 : 代币的集中程度,是不是少数人拥有代币
  • 波动率 : 价格的波动程度

4.1 用户行为指标

4.1.1 活跃地址数量

活跃地址数（Active Addresses），是在特定时间范围内，发生了交易（发送或接收）的唯一钱包地址数量。 这个指标反映了链上用户的活跃程度，可用于评估协议、DApp 或特定代币的用户参与度。长期趋势可以衡量区块链生态的健康度。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    COUNT(DISTINCT "from") + COUNT(DISTINCT "to") AS active_addresses
FROM ethereum.transactions
WHERE block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.1.2 新增地址数量

新增地址数（New Addresses），是在特定时间范围内，首次进行交易的钱包地址数量。 这个指标反映了新用户进入区块链的情况，可用于衡量项目或协议的用户增长能力。较高的新增地址数表明用户群体在扩展，而下降趋势可能意味着用户增长放缓。

WITH first_tx AS (
   SELECT 
       "from", 
       MIN(block_time) AS first_seen
   FROM ethereum.transactions
   GROUP BY "from"
)
SELECT 
   DATE_TRUNC('day', first_seen) AS day,
   COUNT("from") AS new_addresses
FROM first_tx
WHERE first_seen >= CURRENT_DATE - INTERVAL '30' days
GROUP BY 1
ORDER BY 1;

4.1.3 持仓地址数量

持仓地址数（Holding Addresses），是在特定时间范围内，持有某个代币（余额大于 0）的唯一地址数量。 这个指标反映了代币的持有者分布情况，可用于评估项目的去中心化程度。持仓地址数的增长通常意味着代币的接受度提高，而下降可能表明用户流失或集中度增加。

SELECT 
    DATE_TRUNC('day', evt_block_time) AS day,
    COUNT(DISTINCT to) AS holding_addresses
FROM erc20_ethereum.evt_transfer
WHERE contract_address = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48 -- USDC 合约地址
AND evt_block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.2 资金流动

4.2.1 资金流入（Fund Inflow）

资金流入（Fund Inflow），是在特定时间范围内，流入某个协议或某个地址的资金总量。 这个指标反映了市场对某个协议或资产的兴趣程度。较高的资金流入通常意味着用户对该协议或资产的需求增加，而低资金流入可能表示市场热度下降。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    SUM(value / 1e18) AS total_inflow
FROM ethereum.transactions
WHERE to = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48 --  Uniswap V2 Router
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.2.2 资金流出（Fund Outflow）

资金流出（Fund Outflow），是在特定时间范围内，从某个协议或某个地址流出的资金总量。 这个指标可以反映资金是否在离开某个协议或链上应用，可能意味着用户在提取资金、流动性减少，或市场情绪趋向谨慎。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    SUM(value / 1e18) AS total_outflow
FROM ethereum.transactions
WHERE "from" = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48 -- 目标协议地址
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.2.3 资金净流量（Net Flow）

资金净流量（Net Flow），是在特定时间范围内，资金流入与资金流出的差值，即：

资金净流量 = 资金流入 − 资金流出

该指标用于判断市场趋势。如果资金净流量为正，表示资金流入大于流出，通常被视为市场增长信号；反之，如果净流量为负，可能意味着资金外流，市场情绪较为谨慎。

WITH inflow AS (
    SELECT 
        DATE_TRUNC('day', block_time) AS day,
        SUM(value / 1e18) AS total_inflow
    FROM ethereum.transactions
    WHERE to = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48
    GROUP BY 1
),
outflow AS (
    SELECT 
        DATE_TRUNC('day', block_time) AS day,
        SUM(value / 1e18) AS total_outflow
    FROM ethereum.transactions
    WHERE "from" = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48
    GROUP BY 1
)
SELECT 
    inflow.day,
    inflow.total_inflow,
    outflow.total_outflow,
    (inflow.total_inflow - outflow.total_outflow) AS net_flow
FROM inflow
JOIN outflow ON inflow.day = outflow.day
WHERE inflow.day >= CURRENT_DATE - INTERVAL '30 days'
ORDER BY inflow.day;

4.2.4 巨鲸交易（Whale Transactions）

巨鲸交易（Whale Transactions），是指单笔交易金额超过特定阈值的大额交易。 这一指标通常用于监测大资金的流动，尤其是对市场可能产生重大影响的巨鲸（Whales）行为。例如，大额买入可能是市场信号，而大额卖出可能预示市场波动。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    COUNT(*) AS whale_tx_count,
    SUM(value / 1e18) AS whale_tx_volume
FROM ethereum.transactions
WHERE value / 1e18 > 100  -- 设定巨鲸交易阈值为 100 ETH
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.3 智能合约交互

4.3.1 合约调用次数（Smart Contract Calls）

合约调用次数（Smart Contract Calls），是在特定时间范围内，链上智能合约被调用的总次数。 这个指标反映了智能合约的使用活跃度，可用于评估某个协议或 DApp 的用户交互情况。高调用次数通常表示某个智能合约被广泛使用，低调用次数可能表明用户活跃度降低。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    COUNT(*) AS contract_call_count
FROM ethereum.traces
WHERE to IS NOT NULL -- 确保是合约调用
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.3.2 活跃智能合约（Active Smart Contracts）

活跃智能合约（Active Smart Contracts），是在特定时间范围内，至少被调用过一次的唯一智能合约数量。 这个指标反映了链上智能合约的生态活跃度，可以用于评估新项目的增长情况或老项目的用户留存。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    COUNT(DISTINCT to) AS active_contracts
FROM ethereum.traces
WHERE to IS NOT NULL -- 确保是合约调用
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.3.3 Gas 消耗量（Gas Consumption）

Gas 消耗量（Gas Consumption），是在特定时间范围内，所有智能合约交互消耗的 Gas 总量。 这个指标反映了网络拥堵情况和智能合约的计算需求。高 Gas 消耗可能意味着网络繁忙，或者某些智能合约执行了复杂操作。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    SUM(gas_used) AS total_gas_used
FROM ethereum.traces
WHERE block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.3.4 NFT 铸造量（NFT Minting Volume）

NFT 铸造量（NFT Minting Volume），是在特定时间范围内，新创建（Mint）的 NFT 数量。 这个指标反映了 NFT 市场的活跃程度，可用于分析 NFT 生态的增长情况。

SELECT 
    DATE_TRUNC('day', evt_block_time) AS day,
    COUNT(*) AS nft_mint_count
FROM erc721_ethereum.evt_transfer
WHERE from = '0x0000000000000000000000000000000000000000' -- Minting 交易
AND evt_block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.3.5 DeFi 交互量（DeFi Interactions）

DeFi 交互量（DeFi Interactions），是在特定时间范围内，用户与 DeFi 协议进行交互的次数，包括借贷、流动性提供、质押等。 这个指标可以用来评估 DeFi 生态的活跃度，观察用户是否在持续参与 DeFi 协议。

SELECT 
    DATE_TRUNC('day', block_time) AS day,
    COUNT(*) AS defi_interactions
FROM ethereum.traces
WHERE to IN (
    SELECT address FROM labels.all WHERE   name like  'Uniswap%' -- 选择 Uniswap 相关合约
)
AND block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

4.4 市场表现

4.4.1 TVL（Total Value Locked，锁定总价值）

TVL（Total Value Locked），是 DeFi 协议中锁定的总价值，通常指用户存入协议中的资金（例如流动性池、借贷市场中的存款）。 这个指标反映了 DeFi 生态的整体健康状况和市场信任度。较高的 TVL 表明 DeFi 生态活跃，较低的 TVL 可能意味着资金流出或市场信心下降。

TVL的计算逻辑比较简单，即统计出协议中所有相关代币的数目，再乘以每种代币的价格，最后求和得出。为了得到每一天的各个Pair的代币余额，我们要先整理出所有的交易明细记录：

with token_pairs as (
    select 
        coalesce(k1.symbol, 'AGI') || '-' || coalesce(k2.symbol, 'AGI') as pair_name,
        p.pair,
        p.evt_block_time,
        p.token0,
        p.token1,
        p.stable
    from auragi_arbitrum.PairFactory_evt_PairCreated p
    left join tokens.erc20 k1 on p.token0 = k1.contract_address and k1.blockchain = 'arbitrum'
    left join tokens.erc20 k2 on p.token1 = k1.contract_address and k2.blockchain = 'arbitrum'
),

token_transfer_detail as (
    select date_trunc('minute', evt_block_time) as block_date,
        evt_tx_hash as tx_hash,
        contract_address,
        "to" as user_address,
        cast(value as decimal(38, 0)) as amount_raw
    from erc20_arbitrum.evt_Transfer
    where "to" in (select pair from token_pairs)
        and evt_block_time >= date('2023-04-04')

    union all
    
    select date_trunc('minute', evt_block_time) as block_date,
        evt_tx_hash as tx_hash,
        contract_address,
        "from" as user_address,
        -1 * cast(value as decimal(38, 0)) as amount_raw
    from erc20_arbitrum.evt_Transfer
    where "from" in (select pair from token_pairs)
        and evt_block_time >= date('2023-04-04')
),

token_price as (
    select date_trunc('minute', minute) as block_date,
        contract_address,
        decimals,
        symbol,
        avg(price) as price
    from prices.usd
    where blockchain = 'arbitrum'
        and contract_address in (select distinct contract_address from token_transfer_detail)
        and minute >= date('2023-04-04')
    group by 1, 2, 3, 4
    
    union all
    
    -- AGI price from swap trade
    select date_trunc('minute', block_time) as block_date,
        0xFF191514A9baba76BfD19e3943a4d37E8ec9a111 as contract_address,
        18 as decimals,
        'AGI' as symbol,
        avg(case when token_in_address = 0xFF191514A9baba76BfD19e3943a4d37E8ec9a111 then token_in_price else token_out_price end) as price
    from query_2337808
    group by 1, 2, 3, 4
)

select p.symbol,
    d.block_date,
    d.tx_hash,
    d.user_address,
    d.contract_address,
    d.amount_raw,
    (d.amount_raw / power(10, p.decimals) * p.price) as amount_usd
from token_transfer_detail d
inner join token_price p on d.contract_address = p.contract_address and d.block_date = p.block_date

在上面的交易详情查询结果基础上，我们就可以来统计计算每一天的TVL了。

with date_series as (
    select block_date
    from unnest(sequence(timestamp '2023-04-01 00:00:00', localtimestamp, interval '1' hour)) as tbl(block_date)
),

pool_balance_change as (
    select symbol,
        date_trunc('hour', block_date) as block_date,
        sum(amount_usd) as amount
    from query_2339248
    group by 1, 2
),

pool_balance_summary as (
    select symbol,
        block_date,
        sum(amount) over (partition by symbol order by block_date) as balance_amount,
        lead(block_date, 1, current_date) over (partition by symbol order by block_date) as next_date
    from pool_balance_change
    order by 1, 2
)

select d.block_date,
    p.symbol,
    p.balance_amount
from pool_balance_summary p
inner join date_series d on p.block_date <= d.block_date and d.block_date < p.next_date
order by 1, 2

4.4.2 流通总量 Circulating Supply

流通总量是当前市场中以及持币者掌握的流通的加密货币数量。它与总供应量(Total Supply)不同，它不会将无法交易流通的部分纳入统计，比如被锁定而无法交易的货币数量。由于这部分无法流通的加密货币通常并不会影响其价格，所以流通总量作为代币数量的度量较总供应量更为常用。对于不同的加密货币，其计算方式有所不同。比如一些线性释放的代币，他的供应量随时间增加。又如一些有通缩燃烧机制的代币，我们在计算流通总量的时候就要减去这一部分。这里我们以比特币为例，计算它的当前流通总量。

SELECT 
    DATE_TRUNC('day', evt_block_time) AS day,
    SUM(value) / 1e6 AS circulating_supply
FROM erc20_ethereum.evt_transfer
WHERE contract_address = 0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48 -- USDC 合约地址
AND evt_block_time >= CURRENT_DATE - INTERVAL '30' day
GROUP BY 1
ORDER BY 1;

比特币的流通总量计算逻辑较为简单，从起始周期每个区块产出50枚，每210000个区块进入一个减半周期。基于此，我们可以计算出它当前的流通总量：

SELECT SUM(50/POWER(2, ROUND(height/210000))) as Supply                      
FROM bitcoin.blocks

4.4.3 总市值（Market Capitalization）

总市值（Market Capitalization），是代币的市场总价值，计算公式如下

总市值=代币流通量×市场价格

在股票市场，总市值是指在某特定时间内总股本数乘以当时股价得出的股票总价值。相应的在区块链领域，它是一种加密货币的流通总量(Circulating Supply）乘以该加密货币的价格得出的该加密货币总价值。因此计算总市值的关键是计算出我们刚刚学习的指标-流通总量。当我们计算出了流通总量，再乘以该加密货币的当前价格，就能得出其总市值。

这个指标反映了代币的整体市场规模，并常用于与其他加密资产进行对比。

比特币总市值:

SELECT SUM(50/POWER(2, ROUND(height/210000))) as Supply, 
       SUM(50/POWER(2, ROUND(height/210000)) * p.price) /POWER(10, 9) AS "Market Cap"
FROM bitcoin.blocks
INNER JOIN (
    SELECT price FROM prices.usd_latest
    WHERE symbol='BTC'
        AND contract_address IS NULL
) p ON TRUE

4.4.4 代币持仓分布（Token Distribution）

代币持仓分布（Token Distribution），是衡量代币的集中程度，即代币是否被少数地址控制。 通常会计算前 X% 地址持有的代币占比，例如前 10 个地址持有量、前 100 个地址持有量等。

SELECT 
    SUM(balance) / (SELECT SUM(balance) FROM erc20_ethereum.balances WHERE contract_address = 0xTokenAddress) AS top_10_percentage
FROM (
    SELECT balance FROM erc20_ethereum.balances 
    WHERE contract_address = 0xTokenAddress
    ORDER BY balance DESC
    LIMIT 10
) top_holders;

4.4.5 波动率（Volatility）

波动率（Volatility），是衡量代币价格在一定时间内的变动幅度。 计算方法通常是标准差或最大涨跌幅：

波动率=标准差(每日回报率)

WITH price_data AS (
    SELECT 
        day,
        price
    FROM prices.usd_daily
    WHERE symbol = 'ETH'
)
SELECT 
    STDDEV(price) AS price_volatility
FROM price_data
WHERE day >= CURRENT_DATE - INTERVAL '30' day;
