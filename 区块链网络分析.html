Chapter 12 区块链网络分析

12.0.1 什么是区块链网络分析？

区块链网络分析（Blockchain Network Analysis）是指利用链上数据和交易网络，对用户行为、资金流动、合约交互、地址关联性等进行研究，以揭示区块链生态的结构、趋势和潜在风险。

在区块链中，每个交易都被记录在公共账本上，使得研究者可以通过数据分析，发现资金流向、用户群体、交易模式、去中心化程度等关键信息。

12.0.2 区块链网络分析的核心方向

区块链网络分析可以从多个角度进行，主要包括以下几个核心方向：

12.0.2.1 地址与实体分析（Address & Entity Analysis）

  • 研究单个地址的交易行为（例如是否为巨鲸、交易所、智能合约）。
  • 通过聚类分析识别相同控制主体（如交易所的热钱包）。
  • 分析特定地址与其他地址的交互模式，例如：
    • 巨鲸钱包的资金流动
    • 黑客地址的交易路径
    • 交易所资金进出

示例：

  • 通过分析 ETH 交易所的充提记录，可以推测市场的买卖压力。
  • 追踪 黑客攻击资金流动，确定其是否通过 Tornado Cash 等隐私协议洗钱。

12.0.2.2 交易网络分析（Transaction Network Analysis）

  • 研究交易之间的网络关系，构建 交易流图（Transaction Graph）。
  • 计算交易网络的中心性（centrality），寻找重要的交易枢纽（如交易所、巨鲸）。
  • 观察资金如何在链上流动（资金路径分析）。

示例：

  • 以太坊上 USDT 交易网络中，Tether 官方地址通常是主要的资金源，而交易所地址则是资金的主要汇聚点。
  • 分析 MEV 机器人之间的交易关系，找出套利、三明治攻击等行为模式。

12.0.2.3 资金流动追踪（Fund Flow Analysis）

  • 追踪特定代币或资产在区块链上的流动路径。
  • 识别资金如何从 A 地址流向 B 地址，并分析链上资金趋势。
  • 检测异常资金流，例如：
    • 黑客攻击后的赃款流向（跨链桥、隐私协议、CEX 出金）。
    • 庞氏骗局或非法资金链的资金流动。

示例：

  • 2022 年 FTX 破产后，有人监测 FTX 热钱包的资金流动，发现资金被转移到不同钱包并逐步通过 DEX 变现。
  • 通过 Dune Analytics追踪稳定币流向，分析资金在 DeFi 生态中的流动情况。

12.0.2.4 智能合约交互分析（Smart Contract Interaction Analysis）

  • 研究用户如何与智能合约交互，例如：
    • 交易所存取款合约（CEX Deposit/Withdrawal）
    • DeFi 借贷（Aave、Compound）
    • DEX 交易（Uniswap、SushiSwap）
    • NFT 交易（OpenSea、Blur）
  • 计算智能合约的活跃度、调用次数、Gas 消耗等。

示例：

  • 监测 Uniswap V3的流动性提供者（LP）行为，分析大户如何调整流动性仓位。
  • 分析 Aave 借贷市场的清算交易，研究 MEV 机器人如何优先执行清算。

12.0.2.5 以太坊网络健康度分析（Ethereum Network Health Analysis）

  • 评估整个区块链网络的健康度，包括：
    • 活跃地址数（Active Addresses）
    • 交易量（Transaction Volume）
    • Gas 费趋势（Gas Trends）
    • 区块间隔时间（Block Interval）
    • 矿工/验证者分布（Validator Distribution）
    • DeFi TVL 变化（Total Value Locked）

示例：

  • 在以太坊Gas 费高企时，可以研究是因为 NFT 交易过热，还是 DeFi 交易增加导致拥堵。
  • 在 Polygon 网络上，分析每日活跃用户数（DAU）是否增长，判断其生态发展情况。

12.0.3 如何使用 Dune 进行区块链网络分析？

Dune Analytics 提供丰富的链上数据表，可以用于网络分析：

类别 Dune 数据表 用途
交易分析 ethereum.transactions 统计交易量、活跃地址、Gas 费用
资金流动 ethereum.traces 追踪 ETH 及 ERC20 资金流向
DEX 交易 uniswap_v3_ethereum.swaps 研究 Uniswap V3 交易模式
DeFi 借贷 aave_v2_ethereum.LiquidationCalls 分析 Aave 借贷市场的清算
NFT 交易 opensea_ethereum.trades 研究 OpenSea 上的 NFT 交易情况
合约调用 ethereum.logs 统计智能合约调用量，分析合约活跃度

12.1 获取网络数据

这里获取的事用户的交易网络

with wallet_info as (
  
    select 0x59ABf3837Fa962d6853b4Cc0a19513AA031fd32b as wallet_address
      
),

layer_one_relatinon as 
(
      SELECT
         "from" 
        ,"to" 
      FROM ethereum.transactions
      where ( "from"  in (select wallet_address from wallet_info  group by 1) or "to"  in (select wallet_address from wallet_info  group by 1) )
      and success = True
      and ("value"/1E18)> 1000
      and block_number>=15950923    
),

layer_one_all_address as 
(
  
      select
        layer_one_address
      from 
      (
        select
            "from" as layer_one_address
        from layer_one_relatinon
        group by 1
      
        union all
      
        select
            "to" as layer_one_address
        from layer_one_relatinon
        group by 1
      ) a 
      group by 1
),



layer_one_all_address_type as 
(
  
    select 
        targe_address
        ,relation_address_count
        ,case
            when b.wallet_address is not null then 'Core Node'
            when relation_address_count>100 then 'Large Node'
            else 'Normal Node'
        end as  node_type
    from 
    (
        select
          targe_address
          ,count(distinct relation_address) as relation_address_count
        from 
        (
            SELECT
               "from"  as targe_address
              , "to" as relation_address
            FROM ethereum.transactions
            where    "from"  in (select layer_one_address from layer_one_all_address  group by 1)
            and success = True
            and block_number>=15950923  
            group by 
                "from"
              , "to"
          
            union all
          
            SELECT
               "to" as targe_address
               ,"from"  as  relation_address
            FROM ethereum.transactions
            where "to" in (select layer_one_address from layer_one_all_address  group by 1) 
            and success = True
            and block_number>=15950923  
            group by 
                "from"
              , "to"
        ) a 
        group by 1
    ) a 
    left join wallet_info b
    on a.targe_address = b.wallet_address
    
    group by 1,2,3
),

layer_two_relation as 
(
      SELECT
         "from"
        , "to"
      FROM  ethereum.transactions a
      where  (success = True and block_number>=15950923 and  ("value"/1E18)> 1000)
      and  ( "from" in (select targe_address from layer_one_all_address_type where node_type not in ('Large Node') group by 1) or "to" in (select targe_address from layer_one_all_address_type where node_type not in ('Large Node')  group by 1) )
      group by 
          "from"
        , "to"
),



layer_two_address_all_address as 
(  
  select
    layer_two_address
  from 
  (
    select
        "from" as layer_two_address
    from layer_two_relation
    group by "from"
  
    union all
  
    select
        "to" as layer_two_address
    from layer_two_relation
    group by "to"
  ) a 
  group by 
    layer_two_address
),


all_target_address as 
(
    select
        target_addrss
    from
    (
        select
            layer_two_address as target_addrss
        from layer_two_address_all_address
        
        union all 
        
        select
            layer_one_address  as target_addrss
        from layer_one_all_address
    ) a 
    group by 1
) 


SELECT
     "from" 
    ,"to" 
    ,round(sum( ("value"/1E18)),0) as transfer_eth_amount
    ,count(*) as transfer_eth_count
FROM ethereum.transactions
where ( "from"  in (select target_addrss from all_target_address  group by 1) and "to"  in (select target_addrss from all_target_address  group by 1) )
and success = True
and ("value"/1E18)> 1000
and block_number>=15950923  
group by 1,2
